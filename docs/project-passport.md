# Паспорт проекта

## Общая информация

**Название проекта:** Vizitka

**Тип проекта:** одностраничный персональный сайт-визитка

**Назначение:** демонстрация персональной информации, контактов и ссылок на
социальные сети в рамках учебного проекта.

**Учебный контекст:** проект выполнялся для выпускной квалификационной работы в
Тольяттинской академии управления.

**Год:** 2026

## Цель проекта

Создать простой, понятный и адаптивный сайт-визитку, который можно открыть на
компьютере или мобильном устройстве без установки дополнительных приложений.

## Основные задачи

- разработать структуру одностраничного сайта;
- реализовать визуальное оформление в едином стиле;
- добавить переключение русского и английского языков;
- подготовить блок ссылок для связи с автором;
- обеспечить корректное отображение на разных размерах экранов;
- подготовить проект к размещению в приватном GitHub-репозитории.

## Технологии

- HTML5;
- CSS3;
- JavaScript;
- Google Fonts;
- Git;
- GitHub.

## Архитектура

Проект является статическим сайтом и состоит из трех основных файлов:

- `index.html` отвечает за структуру страницы;
- `styles.css` отвечает за внешний вид, сетку, адаптивность и анимации;
- `script.js` отвечает за переключение языков и настройку ссылок.

Серверная часть, база данных и CMS в проекте не используются.

## Пользовательский сценарий

1. Пользователь открывает сайт.
2. На первом экране видит имя, роль и краткую информацию.
3. При необходимости переключает язык интерфейса.
4. Переходит по одной из ссылок для связи или просмотра социальных сетей.

## Требования к эксплуатации

- современный браузер с поддержкой HTML5, CSS3 и JavaScript;
- доступ к интернету для загрузки внешнего шрифта;
- статический хостинг или любой веб-сервер для публикации.

## Ограничения

- данные редактируются вручную в файле `script.js`;
- административная панель отсутствует;
- форма обратной связи отсутствует;
- проект не хранит персональные данные пользователей.

## Результат

В результате создан статический сайт-визитка, подготовленный для хранения в
приватном GitHub-репозитории и дальнейшей публикации на статическом хостинге.
