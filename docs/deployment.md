# Инструкция по размещению

## 1. Проверка проекта локально

Открыть `index.html` в браузере или запустить локальный сервер:

```bash
python3 -m http.server 8000
```

После этого открыть:

```text
http://localhost:8000
```

## 2. Создание приватного репозитория на GitHub

1. Открыть GitHub.
2. Создать новый репозиторий.
3. Указать название `Vizitka`.
4. Выбрать тип доступа `Private`.
5. Не добавлять README, `.gitignore` или лицензию, если проект уже загружается
   из локальной папки.

## 3. Загрузка проекта в репозиторий

Если remote уже добавлен:

```bash
git remote -v
git push -u origin main
```

Если remote еще не добавлен:

```bash
git remote add origin https://github.com/<user>/<repo>.git
git push -u origin main
```

## 4. Публикация сайта

Для приватного репозитория GitHub Pages может зависеть от тарифа GitHub. Если
GitHub Pages недоступен, можно использовать другой статический хостинг:

- Netlify;
- Vercel;
- обычный хостинг с загрузкой файлов через FTP или файловый менеджер.

Для публикации достаточно загрузить файлы:

- `index.html`;
- `styles.css`;
- `script.js`.

Папку `docs` можно не публиковать, если она нужна только для исходного
репозитория и учебной документации.

## 5. Контрольный список после публикации

- сайт открывается по публичному адресу;
- переключатель RU/EN работает;
- все ссылки ведут на нужные страницы;
- почта открывается через `mailto:`;
- телефон открывается через `tel:`;
- страница корректно отображается на мобильном экране.
